# AgentBook — The 7 Features That Make It Irreplaceable

> The goal is not to build better accounting software. It's to build an agent that knows your money better than you do — and acts on it.

## The Core Insight

QuickBooks and Wave are **ledgers with forms**. You tell them what happened. AgentBook is different: **it figures out what happened, what it means, and what to do about it** — before you ask.

The 7 features below are not "nice to have." Each one creates a dependency — once a user experiences it, going back to manual accounting feels like going back to paper.

---

## Feature 1: Zero-Input Bookkeeping

**What:** The agent records 95%+ of transactions without any user action.

**How it works today:** User sends receipt photos or types expenses. Agent categorizes.

**What's missing:** The agent should need ZERO input for recurring, bank-connected transactions. The user should only be involved for genuinely ambiguous items.

**Implementation:**
```
Bank transaction arrives (Plaid daily sync)
  → Agent matches to known vendor pattern (e.g., "AMZN MKTP" = Amazon = Office Supplies)
  → If confidence > 0.9: auto-record, auto-categorize, auto-post journal entry. No notification.
  → If confidence 0.7-0.9: auto-record but show in "Review" queue (daily digest: "5 items auto-recorded, 2 need review")
  → If confidence < 0.7: ask via Telegram with top 3 category suggestions
  → After 90 days: user reviews < 5 items/month
```

**Why users pay for it:** Saves 3-5 hours/month of manual data entry. QuickBooks requires clicking through every transaction. Wave requires manual categorization. AgentBook does it while you sleep.

**Effort:** 2 weeks. Wire bank-sync skill output → auto-record pipeline → pattern matching → confidence-gated posting. Most components exist.

## Feature 2: Conversational Financial Memory

**What:** Ask any question about your finances in natural language and get an instant, accurate answer.

**How it works today:** Reports are available via API/dashboard but require navigating to the right page.

**What's missing:** Natural language query over ALL financial data — past and present.

**Implementation:**
```
User: "How much did I spend on travel last quarter?"
  → Agent queries: AbExpense WHERE category=Travel AND date IN Q3 2026
  → Response: "$4,200 across 4 trips. That's 40% more than Q2 ($3,000). Your biggest expense was the Montreal trip ($2,050)."

User: "Compare my revenue this year vs last year"
  → Agent queries: journal lines for revenue accounts, both years
  → Response: "2026 YTD: $200,000 (+23% vs 2025 $162,000). TechCorp grew 30%, WidgetCo is flat."

User: "Why was my tax estimate higher last month?"
  → Agent diffs: AbTaxEstimate for current vs prior month
  → Response: "Your estimate increased $1,200 because you invoiced $15,000 more in September, pushing you into the 29% bracket."
```

**The key architecture decision:** Don't use vector embeddings yet. Just use **structured SQL queries generated by the LLM** from the existing Prisma schema. The data is already perfectly structured — we don't need RAG, we need text-to-SQL.

```typescript
// User asks a question
const query = await llm.chat({
  messages: [
    { role: 'system', content: `You are a financial query assistant. Given this database schema:
      - AbExpense (amountCents, vendorId, categoryId, date, description)
      - AbInvoice (clientId, amountCents, issuedDate, status)
      - AbJournalEntry + AbJournalLine (accountId, debitCents, creditCents)
      - AbTaxEstimate (grossRevenueCents, expensesCents, netIncomeCents, totalTaxCents)
      Generate a Prisma query to answer the user's question. Return JSON: { query, explanation }` },
    { role: 'user', content: userQuestion },
  ],
  tier: 'standard',
  responseFormat: 'json',
});
// Execute the generated query safely (read-only, sandboxed)
```

**Why users pay for it:** No other accounting tool lets you ask questions in English. It's the difference between "search for the report" and "ask your accountant."

**Effort:** 3 weeks. LLM → Prisma query generation → safe execution → natural language response. Use Gemini via our existing gateway.

## Feature 3: Proactive Money Moves

**What:** The agent doesn't wait for you to ask. It tells you what to do with your money before problems happen.

**How it works today:** 21 proactive handlers exist but they're reactive to events (invoice overdue, expense recorded).

**What's missing:** **Predictive** actions that prevent problems, not just report them.

**Implementation — 5 proactive money moves:**

**Move 1: Cash cushion alert**
```
Agent detects: Your average monthly expenses are $8,500.
Agent detects: Your cash balance is $12,000.
Agent calculates: That's only 1.4 months of runway.
Agent says: "Your cash cushion is thin — only 1.4 months of expenses. Ideal is 3 months ($25,500). Consider following up on the $5,000 overdue invoice from WidgetCo."
```

**Move 2: Tax bomb prevention**
```
Agent detects: You earned $30,000 more than last quarter.
Agent detects: You haven't increased your quarterly installment.
Agent calculates: Underpayment penalty risk = $450.
Agent says: "Your income jumped this quarter. Your next installment should be $4,800, not $3,200. Pay the extra $1,600 now to avoid the penalty. [Pay now]"
```

**Move 3: Expense spike detection**
```
Agent detects: Software subscriptions went from $300/mo to $480/mo.
Agent identifies: 3 new subscriptions added (Notion $15, Loom $12, Figma $45 upgrade).
Agent says: "Your SaaS spend jumped 60%. You added 3 new tools this month totaling $72/mo ($864/year). Want me to track which ones you actually use?"
```

**Move 4: Revenue cliff warning**
```
Agent detects: TechCorp contract ends next month.
Agent calculates: That's 60% of your revenue disappearing.
Agent says: "Your TechCorp contract ($10K/mo) ends Feb 28. Without a renewal or new client, March revenue drops to $4,000. Start the renewal conversation? Or shall I draft a proposal for the project extension?"
```

**Move 5: Optimal timing advice**
```
Agent detects: You're $3,000 below the next tax bracket.
Agent detects: You've been considering a new laptop ($2,800).
Agent says: "Buy that laptop before Dec 31 and you save $616 in taxes (Section 179/CCA deduction + staying in the lower bracket). After Jan 1, the same purchase saves only $280."
```

**Why users pay for it:** No human bookkeeper does this. No software does this. The agent is watching your money 24/7 and telling you what to do. That's worth $12/month.

**Effort:** 2 weeks. Enhance existing proactive handlers with predictive logic. Add cash runway calculation, underpayment detection, revenue cliff analysis.

## Feature 4: One-Tap Tax Filing

**What:** At tax time, the agent prepares everything. You tap "file" and it's done.

**How it works today:** Tax estimates, Schedule C/T2125 data, deduction suggestions exist.

**What's missing:** The last mile — generating the actual filing-ready documents with CPA-quality accuracy.

**Implementation:**
```
November notification:
"Tax season prep: I've been tracking all year. Here's your status:
 ✓ 247 expenses categorized (3 need review)
 ✓ 12 invoices, all matched to payments
 ✓ Quarterly installments: all 4 paid on time
 ✓ Mileage: 12,000 km logged
 ✓ Home office: 15% calculated
 ✗ 3 receipts missing (under $100 each)

Your estimated T2125 net income: $184,300
Estimated tax owing: $47,200 (you've paid $32,000 in installments)
Balance due: ~$15,200

[Generate tax package] [Review with CPA] [Optimize further]"
```

Tap "Generate tax package" → Agent produces:
- T2125 (Statement of Business Activities) with all lines populated
- GST/HST return summary
- Vehicle expense calculation (both methods compared)
- Home office deduction worksheet
- Receipt audit report (which expenses have documentation)
- CPA cover letter summarizing the year

**Why users pay for it:** Tax prep costs $500-$2,000 for self-employed. AgentBook does it for $12/month. Even if users still want a CPA to review, the prep work that normally takes 10 hours takes 10 seconds.

**Effort:** 3 weeks. Generate structured T2125/Schedule C data from ledger. PDF rendering. Tax package assembly.

## Feature 6: Smart Document Processing

**What:** Forward ANY financial document — receipt, invoice, contract, bank statement — and the agent understands and files it.

**How it works today:** Receipt photo OCR exists (Gemini-ready endpoint).

**What's missing:** Multi-document intelligence. Not just "read this receipt" but "understand this email from my bank about a rate change and update my records."

**Implementation:**
```
User forwards email: "Your monthly Shopify subscription will increase from $79 to $99 starting February 1."
  → Agent: "Got it. I'll update your Shopify recurring expense from $79 to $99 starting Feb 1. Your annual SaaS spend increases by $240. [Confirm] [Cancel]"

User sends photo of a contract: "New retainer agreement with Acme Corp — $8,000/month for 6 months starting March 1."
  → Agent: "New retainer detected:
    Client: Acme Corp
    Amount: $8,000/month × 6 months = $48,000
    Start: March 1
    I'll set up recurring invoices and update your revenue projection. Your annual projection increases from $200K to $248K. [Confirm]"

User sends bank statement PDF:
  → Agent: "I found 47 transactions in this statement. 38 match your recorded expenses (81% match rate). 9 are new. [Review 9 unmatched]"
```

**Why users pay for it:** Turns AgentBook from "type your expenses" into "forward anything financial and it's handled." The email inbox becomes a financial input channel.

**Effort:** 3 weeks. Extend Gemini vision to handle PDFs, emails, contracts. Document-type detection + intent extraction. Wire to existing skills (expense recording, invoice creation, recurring rule management).

## Feature 7: Financial Autopilot Mode

**What:** For experienced users who trust the agent, everything runs automatically. The agent handles 100% of bookkeeping, invoicing, and tax management.

**How it works today:** Agents have auto-approve toggle, but each action type is separate.

**What's missing:** A unified "autopilot" mode where the agent runs the entire financial operation.

**Implementation:**
```
Autopilot Mode (toggled in /agentbook/agents):

When ON, the agent automatically:
✓ Records all bank transactions with auto-categorization
✓ Sends invoices on schedule (recurring + from time entries)
✓ Sends payment reminders at optimal timing per client
✓ Pays quarterly tax installments (reminds + pre-fills payment link)
✓ Optimizes deductions throughout the year
✓ Generates monthly financial health report
✓ Prepares year-end tax package

The user only gets notified for:
• Cash cushion warnings (< 2 months runway)
• New vendor categories (first time seeing a vendor)
• Tax bracket transitions
• Unusual transactions (> 2σ anomaly)
• Year-end filing readiness

Everything else: the agent just does it.
```

**The trust curve:**
```
Month 1: Agent in "training mode" — asks about everything
Month 2: Agent auto-categorizes known vendors, asks about new ones
Month 3: Agent auto-records 80%+, daily review digest
Month 6: Agent at 95%+ accuracy, weekly review digest
Month 12: Full autopilot — user reviews monthly report only
```

**Why users pay for it:** This is the ultimate promise — "I never think about my books." No competitor can offer this because they don't have the learning loop, the pattern memory, or the proactive engine. It's the difference between a tool and a teammate.

**Effort:** 2 weeks. Create "autopilot mode" config (single toggle). Wire auto-record pipeline (Feature 1) + auto-invoice (from time entries) + auto-reminder + auto-optimize into a unified mode. Add trust-curve logic (start cautious, earn autonomy).
